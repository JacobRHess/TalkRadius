Security, by design

If you care how your voice gets where it's going — so do we.

Built on standard, audited cryptography. No proprietary magic.

End-to-end encrypted calls

Every voice connection uses WebRTC with DTLS-SRTP — the same encrypted media stack browsers and iOS use for secure calls. Keys are negotiated directly between the two phones. Our servers hold no key material and see no audio.

True peer-to-peer

Once two phones agree to talk, audio flows directly between them over a P2P WebRTC session. The signaling server only helps the phones find each other, then steps out of the way.

Ephemeral identity

No accounts. No email. No phone number. Each session gets a throwaway peer ID that the server generates and discards when you disconnect. Nothing ties one session to the next.

Coarse location only

Your phone shares a ~150m geohash cell with the server so nearby users can discover you. Precise GPS coordinates stay on-device and are used only to render the local map.

Nothing persisted

Calls are never recorded. Session state lives in server memory only and is wiped on disconnect. There is no database, no logs of what was said, nothing to leak later.

Hardened signaling

Strict origin allowlisting, per-IP connection caps, per-connection rate limits, server-issued peer IDs to prevent spoofing, and bounded message sizes. And again — no audio ever touches it.
